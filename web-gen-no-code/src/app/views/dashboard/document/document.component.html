Chào bạn đến với trang quản trị API, thật may mắn khi bạn đã đến được đây

1. Chuyển đỗi cơ sở dữ liệu thành APIs:

2. Cách cấu hình tài khoản - nhóm quyền:

3. Cách cấu hình các api đã được tạo:

  • Ứng dụng chỉ hỗ trợ truy vấn PostgreSQL
  • Nếu cấu lệnh tạo cở sở dữ liệu hợp lệ thì các APIs tương ứng sẽ được tạo tại mục "APIs"
  • Nếu bạn muốn chuyển đỗi lại vui lòng thay đỗi truy vấn bên dưới và ấn nút "Chuyển đỗi lại" (Lưu ý: Dữ liệu và các API đã được tạo sẽ bị xoá)

4. Danh sách API trong hệ thống

Các thông tin cần biết khi cấu hình các APIs:

  • Danh sách API (Prefix: http://host/api/v1/app/2024/schema) .
  • Bạn có thể cài đặt các cấu hình như: kích hoạt, thay đổi phạm vi sử dụng, giới hạn các trường trả về.
  • Tài liệu kỹ thuật của từng APIs đã được viết rất chi tiết - bạn có thể tham khảo và sử dụng cho ứng dụng Front-End.
  • Xem chi tiết hướng dẫn tại Github

6. Danh sách nhóm quyền trong hệ thống

Bạn nên tạo các nhóm quyền phù hợp với hệ thống:

  • Mạc định các quyền khi tạo sẽ được kích hoạt
  • Sau khi tạo các quyền thành công bạn cần khai báo các APIs phù hợp với nhóm quyền bạn đã tạo
  • Lưu ý: Bạn nên tạo một Quyền mạc định khi tạo tạo tài khoản - hệ thống sẽ tự gán quyền đó vào tài khoản được tạo

7. Danh sách Tài khoản trong hệ thống

Danh sách tài khoản này bao gồm các tài khoản người dùng đăng ký và tài khoản được người quản trị tạo:

  • Bạn có thể kích hoạt và phân quyền cho tài khoản tại đây
  • Nên cân nhắc khi phân quyền quản trị cho các tài khoản người dùng đăng ký - tùy nghiệp vụ mà hệ thống bạn cần
